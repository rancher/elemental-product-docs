Alle Komponenten in {elemental-product-name} werden mit Kubernetes verwaltet. Im Folgenden wird erläutert, wie man Kubernetes-Ansätze zum Upgrade der Komponenten verwendet.

== {elemental-product-name} Knoten-Upgrade

{elemental-product-name} Knoten werden mit dem {elemental_operator_name} geupgradet. Bitte beziehen Sie sich auf die Dokumentation xref:references/operatorchart-reference.adoc[{elemental_operator_name}] für vollständige Informationen.

Das Upgrade kann entweder über die Kommandozeilenschnittstelle (CLI) oder die Benutzeroberfläche (UI) erfolgen:

* <<_upgrade_via_command_line_interface,Befehlszeilenschnittstelle>>
* <<_upgrade_via_user_interface,User Interface>>

== Upgrade über die Befehlszeilenschnittstelle

Es gibt zwei Möglichkeiten, Knoten für das Upgrade auszuwählen. Über ein Clusterziel, das ALLE Knoten in einem Cluster auswählt, die unserem Selektor entsprechen, oder über den Knoten-Selektor, der Knoten basierend auf den Knoten-Labels auswählt. Der Knoten-Selektor ermöglicht es uns, gezielter mit dem Upgrade umzugehen, während der Cluster-Selektor einfach alle Knoten in einem übereinstimmenden Cluster auswählt.

Das Upgrade eines bestehenden `ManagedOSImage` löst einen neuen Upgrade-Zyklus aus, um das konfigurierte Image (oder die Image-Version) auf alle Zielknoten abzugleichen.

== Upgrade über die Benutzeroberfläche

Um über die Benutzeroberfläche ein Upgrade durchzuführen, müssen Sie im {elemental-product-name}-Menü "Erweitert" gehen und dann auf `Update Groups` klicken.

Wählen Sie einen Namen, wählen Sie Zielcluster aus und entscheiden Sie sich zwischen den beiden Upgrade-Möglichkeiten :

image::upgrade-ui-menu.png[{elemental-product-name} Upgrade-Menü]

[tabs]
======
Über die verwaltete OS-Version::
+
====
In diesem Fall wird ein `OS Version Channels` verwendet, um die `OS Versions`-Ressourcen automatisch zu befüllen. 

Der untenstehende Kanal wird von uns bereitgestellt, aber Sie können auch Ihren eigenen Kanal mitbringen. 

Siehe Abschnitt <<_managing_available_versions,Verfügbare Versionen verwalten>>, um zu verstehen, wie die `ManagedOSVersion` verwaltet werden. 

image:upgrade-ui-version-channel.png[OS-Version-Kanal erstellen] 

Nach einer kurzen Synchronisationszeit wird Ihr `OS Versions` im `OS Versions`-Menü angezeigt. 

image:upgrade-ui-os-version.png[{elemental-product-name} OS-Version-Menü] 

Schließlich können Sie das `OS Versions` auswählen, wenn Sie Ihr `Upgrade Group` gemäß dem folgenden Screenshot erstellen : 

image:upgrade-ui-upgrade-group-channel.png[OS-Version in Upgrade-Gruppe auswählen]
====

Tab über Bild aus der Registry::
+
====
Geben Sie einfach ein OCI-Image im `Image path`-Feld an, auf das geupgradet werden soll: image:upgrade-ui-image-registry.png[Upgrade via Image Registry]
====
======

Klicken Sie auf die `Create`-Schaltfläche, um den Upgrade-Prozess zu starten. Wenn Sie mehrere Knoten haben, wird das Upgrade nacheinander Knoten für Knoten durchgeführt.
