= Kubernetes 集群配置
:revdate: 2025-11-06
:page-revdate: {revdate}

Kubernetes 集群部署阶段的目标是使用可用的 xref:references/machineinventory-reference.adoc[MachineInventories] 创建一个新的 RKE2 或 K3s 集群，即成功完成 xref:rancher-os-management/architecture/services/architecture-machineonboarding.adoc[Machine onboarding] 阶段的主机。

{elemental-product-name} Kubernetes 集群配置包括以下步骤：

* 用户创建一个 xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate] 资源：该资源允许定义一个 _selector_，根据标签值来识别可用 xref:references/machineinventory-reference.adoc[MachineInventories] 的子集。
* 用户定义一个 xref:references/cluster-reference.adoc[Rancher cluster]，并在 `machinePools` 定义中添加对前一步创建的 xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate] 的引用。
* https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[Rancher RKE2/K3s 集群配置] 对 Rancher 集群资源的创建作出反应，生成数量等于 _quantity_ 的 xref:references/machineinventoryselector-reference.adoc[MachineInventorySelectors] 资源，这个数量在 _machinePools_ 中指定。
* {elemental-product-name} Operator 将每个生成的 xref:references/machineinventoryselector-reference.adoc[MachineInventorySelector] 资源与一个可用的 xref:references/machineinventory-reference.adoc[MachineInventory] 配对，并在由 xref:references/machineinventory-reference.adoc[MachineInventory] 跟踪的主机上安装 https://github.com/rancher/system-agent[rancher-system-agent] 守护程序。
https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[Rancher RKE2/K3s 集群配置] 接管 K3s/RKE2 集群配置，使用 https://github.com/rancher/system-agent[rancher-system-agent] _plans_：它安装所需的组件（例如，containerd、K3s、...），并创建配置文件，直至新 Kubernetes 集群成功部署。
