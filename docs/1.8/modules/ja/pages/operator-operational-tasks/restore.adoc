= 復元
:revdate: 2025-09-08
:page-revdate: {revdate}

このガイドに従って、Rancherを使用してバックアップから{elemental-product-name}設定を復元してください。

== rancher-backupオペレーターとバックアップファイルを復元のために準備します。

公式のhttps://docs.ranchermanager.rancher.io/how-to-guides/new-user-guides/backup-restore-and-disaster-recovery/restore-rancher[Rancherドキュメント]にアクセスし、`rancher-backup operator`がインストールされていてバックアップファイルにアクセスできることを確認してください。

最初にこのxref:operator-operational-tasks/backup.adoc[ドキュメント]に従って、{elemental-product-name}リソースのバックアップを行う必要があります。

== rancher-backupオペレーターを使用して{elemental-product-name}設定を復元します。

バックアップ用tarballを復元するために、必要に応じた`restore object`を作成します。

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Restore
metadata:
  name: elemental-restore
spec:
  backupFilename: rancher-backup-430169aa-edde-4a61-85e8-858f625a755b-2022-10-17T05-15-00Z.tar.gz
----

Kubernetesにマニフェストを適用します。

[,shell]
----
kubectl apply -f elemental-restore.yaml
----

rancher-backupオペレーターのログを確認します。

[,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
----

バックアップファイルが正常に復元されたか確認します。

[,shell]
----
...
INFO[2022/10/31 06:34:50] Processing controllerRef apps/v1/deployments/rancher
INFO[2022/10/31 06:34:50] Done restoring
...
----

https://docs.ranchermanager.rancher.io/how-to-guides/new-user-guides/backup-restore-and-disaster-recovery/migrate-rancher-to-new-cluster[Rancherドキュメント]から手続きを続けます。
